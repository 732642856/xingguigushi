# 🎬 AI 剧本写作系统方案

## 系统概述

一个专为非技术人员设计的剧本创作工作流系统，通过分步骤引导，将您的创意转化为完整的视觉作品。

---

## 🎯 核心流程

```
您的想法
    ↓
[第1步] 生成剧本大纲 → 您确认/修改
    ↓
[第2步] 角色人物侧写 → 您确认/修改
    ↓
[第3步] 生成完整剧本 → 您确认/修改
    ↓
[第4步] 生成分镜头脚本 → 您确认/修改
    ↓
[第5步] 生成 ComfyUI 提示词 → 导入生成画面
```

---

## 📋 详细步骤说明

### 第 1 步：剧本大纲生成

**您的输入：**
- 主题：例如 "一个关于时间旅行的爱情故事"
- 类型：短剧 / 电视剧 / 电影 / 宣传片
- 时长：例如 "15分钟" / "90分钟" / "30集×45分钟"

**AI 输出：**
```
《时光之吻》剧本大纲

【故事梗概】
科学家林墨发明了时间机器，却意外失去了挚爱...

【核心冲突】
改变过去 vs 接受现实

【三幕结构】
第一幕（0-25%）：发现时间机器...
第二幕（25-75%）：多次尝试拯救...
第三幕（75-100%）：最终抉择...

【情感弧线】
希望 → 绝望 → 领悟 → 释然
```

**您的操作：** 阅读 → 说"继续" 或 "修改：把结局改成..."

---

### 第 2 步：角色人物侧写

**基于大纲，AI 生成：**

```
【林墨 - 男主角】
- 年龄：32岁
- 职业：物理学家
- 性格：理性但内心温柔，有执念
- 外貌：戴眼镜，略显疲惫但眼神坚定
- 服装风格：实验室白大褂/休闲衬衫
- 关键道具：怀表（妻子遗物）

【苏婉 - 女主角（回忆/穿越场景）】
- 年龄：28岁
- 职业：钢琴教师
- 性格：乐观开朗，相信缘分
- 外貌：长发，常穿淡色连衣裙
- 标志性动作：弹奏时微微侧头
```

**您的操作：** 确认 或 说"林墨应该更年轻一些"

---

### 第 3 步：完整剧本生成

**AI 输出专业剧本格式：**

```
《时光之吻》

场景 1 - 实验室 - 夜

林墨盯着闪烁的仪器屏幕。

林墨
（喃喃自语）
如果再给我一次机会...

他按下红色按钮，光芒吞没了他。

场景 2 - 公园 - 日（三年前）

阳光透过树叶。苏婉坐在长椅上弹奏 portable piano。

林墨从远处走来，眼中含泪...

[完整对话和动作描述]
```

**您的操作：** 阅读各场景 → 确认 或 修改特定场景

---

### 第 4 步：文字分镜头脚本

**将剧本转化为拍摄指南：**

```
镜头 1
场景：实验室
景别：全景 → 特写
画面：林墨背影，缓缓转身面对镜头
时长：5秒
参考情绪：孤独、决心

镜头 2
场景：公园回忆
景别：中景
画面：苏婉侧脸，阳光洒在她头发上，林墨在背景虚化处
时长：8秒
参考情绪：温暖、怀旧
```

## 🛠️ 实现方案推荐

### 方案 A：最简单（推荐初学者）

**工具：Notion + ChatGPT/Claude**

1. 在 Notion 创建 5 个页面（对应 5 个步骤）
2. 在 ChatGPT 中逐步提问，复制结果到 Notion
3. 最后一步使用专门的 ComfyUI 提示词生成器

**优点：** 零成本，易上手
**缺点：** 需要手动复制粘贴

---

### 方案 B：半自动化

**工具：飞书多维表格 + AI 插件**

1. 创建表格，列：步骤 | 输入 | 输出 | 状态
2. 使用飞书的 AI 功能或连接 Claude API
3. 每个步骤完成自动流转到下一步

**优点：** 有流程管理
**缺点：** 需要学习飞书操作

---

### 方案 C：专业级（推荐）

**工具：Dify / Coze / 扣子（可视化 AI 工作流平台）**

**为什么选择这个：**
- 🎨 可视化拖拽界面，不用写代码
- 🔗 可以连接多个 AI 步骤，自动流转
- 💾 自动保存每次创作记录
- 🚀 可以直接导出 ComfyUI 格式

**搭建步骤：**
1. 注册 Dify（dify.ai）或 Coze（coze.cn）
2. 创建工作流，添加 5 个"LLM 节点"
3. 每个节点配置对应的提示词
4. 添加"问答节点"收集您的确认
5. 导出 API 或直接使用

## 📝 核心提示词模板（可直接使用）

### 大纲生成提示词

```
你是一位资深编剧。根据以下信息生成剧本大纲：

主题：{用户输入的主题}
类型：{短剧/电视剧/电影/宣传片}
时长：{时长}

要求：
1. 故事梗概（200字以内）
2. 核心冲突（主要矛盾）
3. 三幕结构（开端/发展/高潮结局的比例）
4. 情感弧线（主角的情绪变化）
5. 关键场景（3-5个核心场景描述）

风格：{根据类型调整，如短剧要求节奏快、反转多}
```

### 角色侧写提示词

```
基于以下剧本大纲，为主要角色创建详细人物侧写：

大纲：{上一步的大纲}

每个角色包含：
- 基本信息（姓名/年龄/职业）
- 性格特点（3个关键词+详细描述）
- 外貌特征（便于视觉化）
- 服装风格
- 标志性动作或道具
- 角色弧光（在故事中的成长变化）
```

### 剧本生成提示词

```
基于大纲和角色设定，创作完整剧本：

大纲：{大纲内容}
角色：{角色侧写}

格式要求：
- 使用标准剧本格式
- 场景标题：场景编号 - 地点 - 时间
- 角色名大写居中
- 对话前空一行
- 动作描述用括号或单独段落

要求：
- 符合时长要求（估算每分钟约1-1.5页剧本）
- 对白自然，符合角色性格
- 场景转换流畅
```

### 分镜头提示词

```
将以下剧本转化为分镜头脚本：

剧本：{完整剧本}

每个镜头包含：
- 镜头编号
- 场景地点
- 景别（特写/近景/中景/全景/远景）
- 镜头运动（固定/推/拉/摇/移）
- 画面描述
- 参考情绪/氛围
- 预估时长

注意视觉叙事，强调可以通过画面表现的内容。
```

### ComfyUI 提示词生成

```
为以下分镜头生成 ComfyUI 可用的提示词：

分镜头：{分镜头内容}

输出格式（YAML）：
镜头编号:
  场景描述: "简要描述"
  正向提示词: |
    英文提示词，包含：
    - 主体描述
    - 环境/背景
    - 光影效果
    - 摄影风格（35mm film, cinematic lighting等）
    - 画质关键词（8k, photorealistic, high quality）
  负向提示词: |
    需要避免的内容
  建议参数:
    采样器: "dpmpp_2m"
    步数: 30
    CFG: 7-8

针对 z-image 和 ltx2.3 优化：
- 强调摄影级真实感
- 包含电影灯光关键词
- 注意人物一致性（如有多个人物镜头）
```

## 🎬 使用示例

### 示例 1：品牌宣传片

**您的输入：**
- 主题：一款新型降噪耳机的科技与自然融合
- 类型：宣传片
- 时长：60秒

**系统输出流程：**
1. 大纲 → 城市噪音 → 戴上耳机 → 置身森林 → 产品展示
2. 角色 → 都市白领（主角）、自然精灵（意象化角色）
3. 剧本 → 15个场景，无对白，纯视觉叙事
4. 分镜 → 每个镜头3-4秒，强调对比
5. 提示词 → 都市冷色调 vs 自然暖色调对比

### 示例 2：短剧

**您的输入：**
- 主题：误入平行世界的快递员
- 类型：短剧
- 时长：10分钟

**系统输出：**
1. 大纲 → 3幕结构，每幕3分钟左右
2. 角色 → 快递员小李、平行世界的"自己"
3. 剧本 → 快节奏对白，悬念设置
4. 分镜 → 15-20个镜头
5. 提示词 → 现实世界日常感 vs 平行世界奇幻感

## 💡 进阶建议

### 保持一致性的技巧

1. **角色锁定：** 在提示词中加入"参考角色设定：[角色描述]"，确保 AI 不偏离
2. **视觉风格统一：** 在 ComfyUI 提示词中加入统一的关键词（如 "shot on Sony A7IV"）
3. **Seed 固定：** 在 ComfyUI 中使用相同 seed 生成同一场景的不同镜头

### 效率提升

- 创建一个"素材库"文档，记录成功的提示词和参数
- 为不同类型（恐怖/喜剧/爱情）准备不同的风格预设
- 使用模板快速启动新项目

## ❓ 常见问题

**Q: 我不懂技术，能操作吗？**
A: 可以！推荐使用"方案 A"（Notion + ChatGPT），只需要复制粘贴。

**Q: 生成的内容我不满意怎么办？**
A: 每步都可以说"修改"，告诉 AI 具体哪里不对，直到满意再进入下一步。

**Q: 可以用中文写剧本吗？**
A: 完全可以！但 ComfyUI 提示词建议用英文（AI 图像模型对英文理解更好）。

**Q: 如何确保 AI 生成的角色形象一致？**
A: 在第 2 步详细描述外貌，第 5 步的提示词中加入相同的外貌关键词。
