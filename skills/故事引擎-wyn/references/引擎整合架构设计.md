# 星语引擎整合架构设计

> 本文档定义引擎间的数据流转标准、接口规范和整合策略  
> 核心原则：**高内聚、低耦合、标准化接口**

---

## 1. 数据流转标准

### 1.1 统一数据格式 (Engine I/O Protocol)

所有引擎必须遵循以下输入/输出格式：

#### 输入格式 (EngineInput)

```typescript
interface EngineInput {
  // 基础信息
  userId?: string;              // 用户ID (匿名模式可为空)
  timestamp: number;            // 请求时间戳
  
  // 内容输入
  content: {
    text?: string;              // 自由文本输入
    bio?: CharacterBio;         // 结构化小传
    hexagrams?: number[];       // 选中的卦象ID (1-64)
    options?: Record<string, any>; // 引擎特定选项
  };
  
  // 上下文
  context?: {
    previousResults?: EngineOutput[];  // 前置引擎输出
    userPreferences?: UserPrefs;       // 用户偏好
    storyParams?: StoryParams;         // 故事参数
  };
}

interface CharacterBio {
  name: string;
  age: number;
  gender: 'male' | 'female';
  era: string;                  // 时代背景
  occupation: string;           // 职业
  want?: string;                // 欲望
  need?: string;                // 需求
  ghost?: string;               // 创伤
  lie?: string;                 // 谎言
  flaw?: string;                // 缺陷
}

interface StoryParams {
  genre: '悬疑' | '权谋' | '情感' | '史诗' | '都市';
  length: 'short' | 'medium' | 'long';  // 5000/10000/20000字
  perspective: 'protagonist' | 'antagonist' | 'bystander';
  style: '黑色幽默' | '正剧' | '喜剧' | '悲剧';
  ending: '喜剧' | '悲剧' | '开放式';
}
```

#### 输出格式 (EngineOutput)

```typescript
interface EngineOutput {
  // 元数据
  engineId: string;             // 引擎标识
  version: string;              // 引擎版本
  executionTime: number;        // 执行时间(ms)
  
  // 核心输出
  result: {
    // 编剧六维度 (所有引擎必须输出)
    want: string;               // 表面欲望
    need: string;               // 真实需求
    ghost: string;              // 过去创伤
    lie: string;                // 自我欺骗
    flaw: string;               // 性格缺陷
    arc: string;                // 成长弧线
    
    // 引擎特定输出 (可选)
    personality?: string;       // 性格描述 (紫微)
    strategy?: string;          // 策略路径 (奇门)
    clues?: string[];           // 线索列表 (六壬)
    relationship?: string;      // 关系分析 (星宿)
    structure?: StoryStructure; // 故事结构 (64卦)
  };
  
  // 关联资源
  resources?: {
   爽点?: string[];             // 相关爽点桥段ID
    悬念?: string[];             // 相关悬念手法ID
    导演视角?: string[];         // 推荐导演视角ID
    世界观维度?: string[];       // 世界观标签
  };
  
  // 置信度与质量
  quality: {
    confidence: number;         // 0-1 置信度
    completeness: number;       // 0-1 完整度
    coherence: number;          // 0-1 连贯性
  };
  
  // 调试信息
  debug?: {
    logs: string[];             // 执行日志
    intermediate: any;          // 中间结果
  };
}

interface StoryStructure {
  acts: {
    act1: Act;                  // 第一幕 (0-25%)
    act2a: Act;                 // 第二幕A (25-50%)
    act2b: Act;                 // 第二幕B (50-75%)
    act3: Act;                  // 第三幕 (75-100%)
  };
  beats: Beat[];                // 节拍点
  turningPoints: TurningPoint[];// 转折点
}

interface Act {
  summary: string;              // 幕摘要
  scenes: Scene[];              // 场景列表
  wordCount: number;            // 字数
}

interface Scene {
  id: string;
  title: string;
  content: string;
  characters: string[];
  location: string;
  time: string;
}
```

### 2.2 引擎协作协议

#### 模式A：顺序依赖 (Pipeline)

```javascript
// 紫微 → 64卦 (人物确定后再定结构)
const result = await pipeline(
  userInput,
  [ziweiEngine, hexagramEngine]
);

// 内部实现：前一个引擎的输出作为后一个的context
async function pipeline(input, engines) {
  let context = {};
  for (const engine of engines) {
    const output = await engine.execute({
      ...input,
      context: { previousResults: context }
    });
    context[engine.id] = output;
  }
  return context;
}
```

#### 模式B：并行融合 (Parallel)

```javascript
// 所有引擎同时执行，结果融合
const results = await Promise.all([
  ziweiEngine.execute(input),
  qimenEngine.execute(input),
  daliurenEngine.execute(input),
  nakshatraEngine.execute(input)
]);

const fused = await fusionEngine.merge(results);
```

#### 模式C：主从增强 (Master-Slave)

```javascript
// 主引擎生成主体，辅助引擎提供增强
const mainResult = await hexagramEngine.execute(input);

const enhancements = await Promise.all([
  mysteryEngine.generateClues(mainResult),    // 六壬生成线索
  worldviewEngine.enrich(mainResult),          // 世界观深化
  writingResources.embedHighlights(mainResult) // 爽点嵌入
]);

const final = { ...mainResult, enhancements };
```

## 4. 冲突解决机制

### 4.1 数据冲突

当多个引擎输出冲突时，按优先级仲裁：

### 4.2 融合策略

```javascript
class FusionEngine {
  // 加权平均融合
  weightedMerge(outputs, weights) {
    const result = {};
    for (const key of Object.keys(outputs[0].result)) {
      const values = outputs.map(o => o.result[key]).filter(Boolean);
      if (values.length > 0) {
        result[key] = this.mergeValues(values, weights);
      }
    }
    return result;
  }
  
  // 互补融合 (不冲突字段合并)
  complementaryMerge(outputs) {
    const result = {};
    for (const output of outputs) {
      for (const [key, value] of Object.entries(output.result)) {
        if (!result[key]) {
          result[key] = value;
        }
      }
    }
    return result;
  }
  
  // 优先级融合 (按引擎优先级覆盖)
  priorityMerge(outputs, priorityOrder) {
    const sorted = outputs.sort((a, b) => {
      return priorityOrder.indexOf(a.engineId) - priorityOrder.indexOf(b.engineId);
    });
    
    const result = {};
    for (const output of sorted) {
      Object.assign(result, output.result);
    }
    return result;
  }
}
```

## 5. 性能优化

### 5.1 缓存策略

```javascript
class EngineCache {
  constructor() {
    this.cache = new Map();
    this.ttl = 30 * 60 * 1000; // 30分钟
  }
  
  getKey(input) {
    // 基于输入特征生成缓存键
    return hash(JSON.stringify(input));
  }
  
  async getOrExecute(key, executor) {
    if (this.cache.has(key)) {
      const { value, timestamp } = this.cache.get(key);
      if (Date.now() - timestamp < this.ttl) {
        return value;
      }
    }
    
    const result = await executor();
    this.cache.set(key, { value: result, timestamp: Date.now() });
    return result;
  }
}
```

### 5.2 懒加载

```javascript
// 引擎动态加载
async function loadEngine(engineId) {
  const config = EngineRegistry[engineId];
  const module = await import(config.entry);
  return module.default;
}

// 按需实例化
const engineInstances = new Map();

async function getEngine(engineId) {
  if (!engineInstances.has(engineId)) {
    const Engine = await loadEngine(engineId);
    engineInstances.set(engineId, new Engine());
  }
  return engineInstances.get(engineId);
}
```
