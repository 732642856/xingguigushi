# 故事引擎

文学创作全流程引擎。从灵感到成文，一站式解决。

---

## 核心定位

| 模块 | 能力 |
|---|---|
| **灵感工坊** | 点子→梗概→大纲，层层递进 |
| **人物工坊** | 小传生成、关系图谱、道德坐标、人物弧光 |
| **结构工坊** | 三幕结构、多线叙事、悬疑埋点、节奏诊断 |
| **对白工坊** | 角色语言风格、黑色幽默台词、场景对白生成 |
| **内容工坊** | 批量音视频转录、字幕生成、文档转换 |
| **会诊工坊** | 多维度对抗性审查、专家并行会诊、分歧透明化 |

---

## 快速入口

| 用户说 | 执行 |
|---|---|
| "我有个灵感" | 启动灵感→梗概→大纲流程 |
| "设计人物" | 输入人物类型→生成完整小传 |
| "检查结构" | 输入大纲→诊断结构问题 |
| "写段对白" | 输入场景+人物→生成对白 |
| "埋个伏笔" | 输入情节→设计伏笔+回收方案 |
| "批量转录" | 启动音视频转录任务 |
| "生成字幕" | 输入视频→输出SRT/VTT |
| "会诊" / "审查" | 启动会诊工坊，多专家对抗性审查 |

---

## 工作流

> **管线定义**：完整的6阶段创作管线见 `references/workflow-pipeline.yaml`
> 阶段：绿灯评估 → 灵感开发 → 人物工厂 → 结构设计 → 剧本写作 → 剧本会诊
> 门禁为 advisory（建议性），创作者可以跳过，可从任意阶段进入。

### 流程1：灵感工坊（点子→大纲）

**阶段1：灵感收集**
- 输入：零散想法、关键词、题材偏好
- 输出：核心概念卡片

**阶段2：梗概生成**
- 输入：核心概念
- 输出：500字故事梗概（起承转合）

**阶段3：大纲展开**
- 输入：故事梗概
- 输出：
  - 分集大纲（剧集）/ 章节大纲（小说）
  - 主线/支线/伏笔线梳理
  - 关键情节点标记

---

### 流程2：人物工坊

**人物小传模板**：

```markdown
## [角色名]

**道德坐标**：X=[-5~+5] / Y=[-5~+5]
- X轴：手段（阴←→阳）
- Y轴：目的（冷←→热）

**核心信息**
- 年龄/职业/身份：
- 核心矛盾：
- 核心欲望：
- 核心恐惧：

**人物弧光**
- 起点：
- 转折点（第X集）：
- 终点：

**标志性特征**
- 语言习惯：
- 行为特征：
- 物品符号：

**关系网**
- 与角色A：关系+动态
- 与角色B：关系+动态
```

**关系图谱**：
- 生成Mermaid关系图
- 标记冲突关系（红色）
- 标记同盟关系（绿色）
- 标记暧昧关系（虚线）

### 流程3：结构工坊

**三幕结构诊断**：

| 幕 | 占比 | 关键元素 | 检查点 |
|---|---|---|---|
| 第一幕（建置） | 25% | 世界建立、人物出场、日常打破 | 催化剂事件位置？ |
| 第二幕（对抗） | 50% | 上升行动、中点转折、下降行动 | 中点是否翻转？ |
| 第三幕（解决） | 25% | 危机、高潮、结局 | 高潮是否解决主线？ |

**悬疑埋点系统**：

```
伏笔设计模板：

伏笔ID：
埋设位置：第X集/章
表面信息：（观众看到的）
隐藏信息：（真相）
回收位置：第Y集/章
回收方式：（揭示/误导读者的反转）
重要性：核心/辅助/红鲱鱼
```

**多线叙事检查**：
- 主线进度追踪
- 支线A/B交叉点标记
- 伏笔线回收状态

### 流程4：对白工坊

**黑色幽默公式**：

```
场景：[场合/氛围]
角色A身份：[职业/阶层/处境]
角色B处境：[当前遭遇]
讽刺点：[身份反差/场合错位/期望落差]

→ 生成对白
```

**对白示例**：

```
场景：追悼会
角色A：丧葬用品销售
角色B：死者家属
讽刺点：商业化的死亡

角色A：（递上名片）这是我的联系方式，以后用得着。
角色B：（接过名片）以后？
角色A：（意识到不妥）我是说...您家里其他人可能...
角色B：（把名片塞回他手里）你先给自己留着吧。
```

**角色语言指纹**：
- 每人独特的口癖、语速、用词习惯
- 阶层差异体现（知识分子vs底层小人物）
- 地域特色（方言、俚语）

**批量转录**：
- 音视频→文字

## 创作风格

**黑色幽默**：
- 用轻松语气说残酷的事
- 身份/场合/期望的反差
- 冷峻底色+温情点缀

**悬疑节奏**：
- 每集至少一个「意外」
- 每集至少一个「揭示」
- 每集至少一个「钩子」

**道德模糊**：
- 没有绝对好人/坏人
- 恶有来路，善有去处
- 不评判，让观众自己判断

### 流程6：会诊工坊（对抗性审查）

**核心理念**：专家的任务是"找问题"，不是"夸好"。

**会诊流程**：

**Step 1 — 维度识别**
系统自动扫描内容，识别涉及哪些审查维度：

| 维度 | 匹配的专家模型 |
|---|---|
| 结构/节奏 | 麦基、救猫咪、Dramatica |
| 人物/弧光 | 北电体系、李安、人物弧光检验 |
| 对白/张力 | 索尔金、塔伦蒂诺、语言指纹 |
| 类型适配 | 该类型的代表导演/编剧 |
| 情感逻辑 | 情感工程、观众共鸣检验 |
| 世界观/细节 | 类型专属知识资产 |

**Step 2 — 并行专家会诊**
- 选择 2-4 个匹配维度，每个维度用对应的导演/编剧模型独立审查
- 每个专家只输出：**问题 + 严重程度 + 修改建议**
- 专家之间不能互相参考，必须独立出意见

**Step 3 — 汇总会诊报告**

```markdown
# 会诊报告

## 共识问题（所有专家都指出的）
1. [问题描述] — 严重程度：P0/P1/P2 — 修改建议：...
2. ...

## 分歧点（专家意见不同的，必须呈现双方理由）
- 专家A认为：...
- 专家B认为：...
- → 由创作者裁决

## 修改建议（按优先级排序）
- P0（必须改）：结构性问题/逻辑硬伤
- P1（建议改）：节奏/人物/对白问题
- P2（锦上添花）：润色/提升
```

**硬性规则**：
1. 分歧不得静默消解——如果两个专家意见不同，必须呈现给创作者
2. 每个问题必须附带具体修改建议，不能只说"这里不好"
3. 审查完成后，P0 问题必须修改才能进入下一阶段
