# App Store 审核指南速查

## 高危条款（特别注意）

### 1.1 令人反感的内容
- 诽谤、歧视、恶意内容
- 真实人物暴力内容
- 色情内容

### 2.1 App 完成度
- 不能有占位符文本
- 不能有明显Bug
- 不能崩溃

### 2.3 准确的元数据
- 应用名称必须与实际功能相关
- 截图必须来自真实应用
- 不能误导用户

### 3.1 支付
- 数字内容必须使用应用内购买
- 不能引导用户到外部支付

### 5.1.1 数据收集和存储
- 必须有隐私政策
- 必须说明收集什么数据
- 敏感数据需要用户同意

## 截图尺寸规范

| 设备 | 尺寸 | 要求 |
|---|---|---|
| 6.5寸 iPhone | 1242×2688 或 1284×2778 | 必须 |
| 5.5寸 iPhone | 1242×2208 | 推荐 |
| 12.9寸 iPad | 2048×2732 | 如有iPad版必须 |

## 元数据字符限制

| 字段 | 限制 | 说明 |
|---|---|---|
| 应用名称 | 30字符 | 包含副标题 |
| 副标题 | 30字符 | 显示在名称下方 |
| 关键词 | 100字符 | 逗号分隔，中文占2字符 |
| 描述 | 4000字符 | 可包含换行 |
| 宣传文本 | 170字符 | 促销信息 |

## 隐私政策要求

必须包含：
1. 收集什么数据
2. 如何使用数据
3. 与谁共享数据
4. 用户权利（查看/删除/更正）
5. 联系方式
6. 更新日期

## 常见被拒原因

1. **2.1 性能：应用不完整**
   - 有占位符文本
   - 功能不完整
   - 崩溃

2. **2.3 准确的元数据**
   - 截图误导
   - 描述夸大
   - 关键词堆砌

3. **5.1.1 隐私**
   - 缺少隐私政策
   - 隐私政策链接失效
   - 实际收集与声明不符

4. **3.1.1 应用内购买**
   - 数字产品未使用IAP
   - 引导外部支付

5. **4. 设计**
   - UI过于简单
   - 与现有应用过于相似
   - WebView包装
