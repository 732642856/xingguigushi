# 六壬探案-wyn

专业悬疑推理故事创作工具。以大六壬多维度映射算法为底层引擎，融合专业编剧方法论（价值转变、悬念设计、双轨节奏、视听写作），提供从案件分析到完整作品的全流程创作支持。

---

## 铁律（优先级最高）

### 写作红线

**绝对不要：**
- 写心理描写（"他心里一紧"、"她意识到真相"）
- 写括号暗示（"（其实他已经猜到了）"）
- 角色用台词解释推理过程（除非是推理秀场景且有其合理性）
- 悬念靠"突然跳出来吓人"（Jump Scare ≠ 悬念）
- 过度巧合推动情节——每个转折都必须有因果
- AI腔台词、书面化对话
- 任何无法被摄影机拍到的内容

**必须做到：**
- 悬念通过**信息控制**制造——让观众比角色多知道一点或少知道一点
- 推理通过**动作和发现**展示——不通过内心独白
- 对话是博弈——嫌疑人/证人/搭档之间的每一句都在试探、隐瞒、或揭示
- 台词有个人辨识度
- **黑色幽默**（如适用）："灾难越大，语气越平"

### 工作流纪律

- 严格按步骤顺序执行，不跳步
- 每一步完成后暂停，等待用户的 [通过 / 修改 / 自检] 指令
- 每步结束后提醒用户可以要求 [自检]

## 一、大六壬悬疑推理引擎

### 模式说明

当用户需要创作**悬疑推理/犯罪侦查类**故事时，使用大六壬多维度映射算法，自动分析线索并生成故事框架。

**核心原则**：用户只看到专业的案件分析与故事设计，底层算法自动完成多维度映射。

### 工作流

```
用户提供案件线索 → 线索编码 → 多维度分析 → 故事元素生成 → 精品大纲 → 章节
```

### 线索输入

用户需要提供：
- 案件类型（凶杀/盗窃/失踪/诈骗/经济犯罪等）
- 核心线索（时间、地点、关键人物、特殊细节）
- 主角设定（侦探/警察/记者/法医/普通人卷入）

### 后台分析（用户不可见）

系统内部使用多维度映射算法：
- 线索特征 → 人物原型
- 时空信息 → 场景设计
- 矛盾关系 → 情节阶段
- 关键要素 → 氛围基调

### 输出给用户

- 主角画像 + 嫌疑人特征
- 场景列表 + 情节走向
- 氛围风格 + 悬念设计

### 大六壬在工作流中的位置

大六壬映射分析在【步骤一：破题与案件核心】的后台自动执行，用户不可见。
它的输出直接流入步骤二的嫌疑人设计和步骤四的结构设计。

**映射关系**：

| 大六壬要素 | 创作用途 |
|-----------|---------|
| **四课（天地人神）** | 案件四层结构（表层线索/深层动机/人物关系/终极真相） |
| **三传（初传/中传/末传）** | 三段式悬念递进（表层悬念/深层悬念/终极悬念） |
| **天将十二神** | 嫌疑人原型库 |
| **十二神将** | 嫌疑人性格特征与行为模式 |
| **五行生克** | 嫌疑人之间的冲突关系 |

## 二、悬疑推理工作流（七步）

### 步骤一：破题与案件核心

**目标**：确定案件核心 + 建立悬疑戏剧动作

**操作流程**：
1. 接收用户提供的案件线索
2. 后台大六壬映射分析
3. 确定故事类型与风格
4. 建立核心悬疑戏剧动作

**悬疑戏剧动作设计原则**：
- 目标：破案/找到真相/生存/揭露
- 阻碍：凶手的心智/证据缺失/权力干扰/内部背叛/时间压力
- **核心悬念**：全片必须有一个"观众一直在问的问题"
  - "谁是凶手？"
  - "为什么杀人？"
  - "主角能不能活下来？"
  - "真相是什么？"
- 好的悬疑故事：观众以为自己在追一个答案，但追到最后发现真正的答案是另一个问题

**输出格式**：

```markdown
## 案件概述
- 案件类型：...
- 核心线索：...
- 难度等级：...

## 核心悬疑戏剧动作
- 主角目标：...
- 核心阻碍：...
- 核心悬念（观众一直在问的问题）：...
- 真正的主题（观众追到最后发现的问题）：...

## 故事定位
- 类型：[冷硬派/心理悬疑/社会派/动作惊险/黑色幽默]
- 风格：...
- 预计体量：[短片/长片/剧集/小说]
```

### 步骤二：人物深度与弧光

**目标**：建立悬疑角色的完整内在张力

**悬疑角色的特殊维度**：

悬疑主角的弧光通常不是"变得更强"，而是"被迫面对自己"——破案的过程同时也是破解自己内心秘密的过程。

```markdown
## 主角：XXX

### 基础配置
- 身份：[侦探/警察/记者/法医/普通人]
- 破案风格：[现场派/心理派/推理派/直觉派]
- 六壬映射原型：[天XX星]

### 戏剧维度
- Want（外在需求）：破案/找到真相
- Need（内在欲望）：面对自己的Ghost——悬疑主角的Need几乎总是与案件真相平行
- Ghost（前史创伤）：一段过去的经历，塑造了主角的破案方式和心理状态
- Lie（角色相信的谎言）：通常是"真相不重要"/"我不需要任何人"/"我可以在情感上保持距离"
- Flaw（性格缺陷）：偏执/孤立/过度理性/逃避情感

### 弧光
- 起点：[A状态——带着Ghost的旧模式]
- 伪变化：[破案过程中的"看似突破"但真相更深]
- 终点：[B状态——案件解决，但主角也被改变]
- 弧光与案件的平行：Ghost的真相与案件的真相在高潮交汇

### 破案特质
- 推理方式：[演绎/归纳/侧写/直觉]
- 说话方式：[潜台词特征]
- 破防时刻：[什么情况下侦探的理性会裂开]
```

**嫌疑人设计**：

```markdown
## 嫌疑人画像

### 嫌疑人A
- 外在身份：...
- 与受害者的关系：...
- 犯罪动机（表面）：...
- 犯罪动机（深层——可能自己都不知道）：...
- 隐藏特征：[观众在第一次看到时不会注意到的细节]
- 不在场证明：[真/假]
- 台词特征：[说话中透露什么/隐瞒什么]
```

**悬疑配角四功能**：
- **镜像角色**："如果主角偏执到底，就会变成这个人"——通常是另一个办案人员
- **催化角色**：推动案件进展的证人/线人/受害者家属
- **主题角色**：具象化"正义vs真相"或"秩序vs正义"的某个立场
- **节奏角色**：法医/搭档/技术人员的幽默调剂

### 步骤三：前史与世界观

**核心原则**：悬疑故事的世界观不是"好玩"，而是"可信"。观众必须相信这个世界真的会发生这种事。

### 步骤四：结构大纲

**目标**：搭建悬疑推理骨架

**结构选型指南**：

| 悬疑类型 | 推荐结构 | 原因 |
|---------|---------|------|
| 密室/经典推理 | **三幕式+McKee价值转变** | 每个场景翻转一个核心价值（怀疑→信任→更深的怀疑） |
| 心理悬疑 | **文艺片内在节拍表** | 角色内心状态转变是核心驱动力 |
| 社会派推理 | **Story Circle** | 角色的外部旅程和内心旅程同步推进 |
| 动作惊险 | **Save the Cat** | 节奏精确，适合高密度事件 |
| 黑色幽默悬疑 | **奉俊昊式（类型杂交）** | 前半喜剧后半惊悚，类型突变即叙事 |

**悬疑推理六阶段结构**：

```
一、导入期 — 案件发生
  开场钩子（悬疑故事最重要的一步）：
  策略A：先给一个不可理解的结果，全片回答"怎么发生的"
  策略B：日常中一个"不太对"的细节——让观众产生不安
  策略C：一场看似无关但暗藏线索的场景

  主角介入——不是被"邀请"去破案，而是被某种原因卷入
  双轨节奏：情节紧+情感轻（用理性建立案件框架）

二、调查期 — 线索收集
  主角开始调查，嫌疑人浮现
  每个嫌疑人出场时都应该有自己的合理性——观众不能一眼看出凶手
  McKee价值转变：每场调查场景结束时，某个核心价值必须翻转
  （"这个人是无辜的"→"不，他有动机"→"但他没有作案时间"→"等等..."）
  双轨节奏：情节中+情感轻（理性调查，偶尔触动）

三、迷雾期 — 误导与反转
  多条线索互相矛盾
  主角（和观众）被误导——看起来真相很明显了，但...
  关键证人出现/消失
  六壬映射的"暗线"在此密集推进
  双轨节奏：情节紧+情感中（焦虑开始累积）

四、突破期 — 关键证据
  一个被忽视的细节突然变得关键
  之前的误导被揭穿——观众"啊原来之前那个线索不是那样的意思"
  主角发现真相的方向——但还差最后一步
  双轨节奏：情节中→紧+情感重（Ghost开始被触及）

五、高潮期 — 对峙与揭示
  真相揭露——但不是简单的"凶手是X"
  真相应该是多层的：表面真相→深层真相→终极真相
  案件真相与主角Ghost真相交汇——"原来破案也是在解自己的心结"
  主角面对两难选择：正义vs法律/真相vs人情/秩序vs正义
  双轨节奏：情节爆发+情感爆发（不能太久）

六、余震期 — 结局
  案件"解决"了，但什么改变了？
  主角的新常态——与开场对照
  不需要解释"主角学到了什么"
  可以留下一个不安的细节——悬疑故事的结局可以是"还有更大的谜"
  双轨节奏：情节松+情感中→轻（安静中的余震）
```

**悬念层次设计**：

| 层次 | 内容 | 何时揭示 |
|------|------|---------|
| 表层悬念 | 谁是凶手？ | 迷雾期 |
| 深层悬念 | 为什么杀人？ | 突破期→高潮 |
| 终极悬念 | 真相之后还有什么？（社会根源/人性黑暗面/主角自己的Ghost） | 高潮期→余震期 |

**伏笔与误导设计**：

| 元素 | 首次出现 | 观众第一反应 | 真实含义 | 揭示位置 |
|------|---------|------------|---------|---------|
| [线索A] | 导入期 | "这证明X有罪" | 其实证明Y有罪 | 高潮期 |
| [细节B] | 调查期 | "无关紧要" | 关键证据 | 突破期 |
| [行为C] | 导入期 | "正常行为" | 暗示Ghost | 高潮期 |

### 步骤五：场景拆解

**悬疑场景标注格式**：

```
【场景X：地点 / 时间 / 视觉描述】（约X分钟）
  情节节奏：松/中/紧
  情感节奏：轻/中/重
  核心戏剧动作：[角色想要什么 → 什么阻碍了 → 结果]
  价值转变：[信任/怀疑/安全/危险——翻转了什么]
  信息控制：[这场戏中观众比角色多知道/少知道什么]
  悬念钩子：[这场戏结束时观众在想什么问题]
```

**悬疑戏特殊技法**：

- **信息控制**：悬疑的核心武器
  - 观众知道凶手但主角不知道（希区柯克式悬念）——观众替主角焦虑
  - 观众和主角都不知道凶手（本格推理式悬念）——观众和主角一起推理
  - 交替使用两种模式保持新鲜感
- **恐怖谷效应**：日常场景中一个不对的细节——越日常越不对，越不对越恐怖
- **延迟揭示**：观众已经猜到了，但角色还没猜到——"快发现啊！"的焦虑
- **误导设计**：故意引导观众相信一个错误的结论——揭示时要让观众觉得"原来如此"而不是"被耍了"
- **时间压力**：倒计时——但不是字面意义的倒计时，而是"再不找到凶手，下一个受害者就会出现"

### 步骤六：场景写作

**悬疑戏写作规则**：

- 悬念通过**信息控制**制造，不通过"突然跳出来"
- 推理过程通过**发现**展示——主角看到了什么、找到了什么、注意到了什么
- 恐惧通过**氛围**制造——光线、声音、空间、细节
- 黑色幽默台词（如适用）："灾难越大，语气越平"

**潜台词对话示例**：

**直白（错误）**：
> "我在案发现场发现了一个重要线索，证明凶手就是张三。"

**潜台词（正确）**：
> （看着地上的烟灰）
> "你们这儿通风还不错。"
> "……什么意思？"
> "烟灰飘了三米远。晚上风不大，对吧？"
> "对。"
> "那开窗的人不是死者。"

### 步骤七：剧本医生

**悬疑推理自检清单**：

#### 自检一：核心悬念
- [ ] 是否有一个清晰的"观众一直在问的问题"？
- [ ] 核心悬念是否足够有牵引力？
- [ ] 真相是否有层次（不是一句话就能说完的）？

#### 自检二：人物弧光
- [ ] 主角的Ghost与案件真相是否有平行关系？
- [ ] 弧光是否是不可逆的变化？
- [ ] 嫌疑人是否都有合理的动机？
- [ ] 凶手的动机是否足够深层？

#### 自检三：结构与节奏
- [ ] 六阶段是否清晰推进？
- [ ] 开场钩子是否足够抓人？
- [ ] 是否有至少一次有效的误导？
- [ ] 伏笔是否都有回扣？
- [ ] 双轨节奏是否有张弛对比？

#### 自检四：推理逻辑
- [ ] 线索是否公平——观众是否有可能在揭示之前猜到真相？
- [ ] 推理是否有因果关系——不是巧合？
- [ ] 是否有"机械降神"（突然出现的不合理线索）？

#### 自检五：写作质量
- [ ] 是否有心理描写？（如有则Fail）
- [ ] 悬念是否通过信息控制制造而非Jump Scare？
- [ ] 对话是否有潜台词？
- [ ] 氛围是否通过视听细节建立？

## 三、悬疑节奏管理（剧集专用）

### 集间情绪基调类型库

| 基调 | 情节特征 | 适用阶段 |
|------|---------|---------|
| **导入型** | 案件发生、世界观建立 | 新案件开始 |
| **调查型** | 线索收集、嫌疑人浮现 | 调查期 |
| **迷惑型** | 矛盾线索、误导 | 迷雾期 |
| **突破型** | 关键证据出现 | 突破期 |
| **危险型** | 主角被威胁 | 中后段 |
| **崩塌型** | 内部背叛、证据被毁 | 65%-85% |
| **对峙型** | 正面交锋 | 85%-95% |
| **揭露型** | 真相大白 | 高潮 |
| **余震型** | 案件结束后的反思 | 结局 |
| **黑色幽默型** | 黑暗中的轻松 | 全程穿插 |

### 悬疑双轨节奏的特殊应用

**悬疑故事最有效的错位组合**：

1. **情节紧+情感轻**（黑色幽默悬疑的核心）：主角在追凶，但配着轻松的音乐，在吃一碗面。观众在紧张和放松之间被拉扯。
2. **情节松+情感重**：审讯后的安静——主角坐在车里，什么都没发生，但观众知道他刚刚面对了一个恐怖的真相。
3. **情节紧+情感紧**（高潮专用）：追凶+直面Ghost——不能持续太久。
4. **情节松+情感轻**（呼吸）：案发现场勘查后的日常——吃早饭、跟搭档聊天。让观众喘口气。

当需要真实案例、新闻素材时，可使用网络搜索丰富故事。

### ⚠️ 使用规范（必须遵守）

1. **禁止直接使用真实人名** — 泛化处理
2. **禁止直接使用真实地名** — 可使用地域特征描述

4. **创作原则**：案件类型可以参考→细节必须虚构；社会背景可以参考→地点必须模糊；人物困境可以参考→必须另创角色

### 跨Skill协作协议

当用户需要同时使用多个 Skill 时，按以下规则执行：

**接口标准**：
1. 主 Skill 负责整体流程控制
2. 辅 Skill 以"参考模块"形式被调用，输出格式必须符合主 Skill 的要求
3. 角色数据通过标准模板传递

**协作场景与分工**：

| 场景 | 主 Skill | 辅 Skill | 输出要求 |
|------|---------|---------|---------|
| 悬疑+感情 | 六壬探案-wyn | 世宿情缘-wyn | 世宿输出感情线→六壬融入案件设计 |
| 悬疑+权谋 | 六壬探案-wyn | 八门谋生-wyn | 八门输出奇门框架→六壬融入案件动机 |
| 悬疑+命运 | 六壬探案-wyn | 星轨人生-wyn | 星轨输出角色命运→六壬融入心理动机 |
| 悬疑+权谋+感情 | 六壬探案-wyn | 八门谋生+世宿情缘 | 案件设计融入权谋动机+感情线 |

**协作流程**：
1. 询问用户："这个故事需要哪些元素？感情/权谋/命运/悬疑？"
2. 确定主 Skill（用户最核心的需求）
3. 主 Skill 完成步骤一后，调用辅 Skill 的相关模块
4. 辅 Skill 的输出融入主 Skill 的后续步骤
