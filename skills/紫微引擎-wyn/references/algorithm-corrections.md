## Bug #1 — 天府星公式错误（最严重，影响所有命盘天府系）

**错误代码：**
```js
tianfuIdx = (12 - ziweiIdx) % 12
```

**问题：** 认为天府+紫微宫位索引之和=12，实际错误。  
当 ziweiIdx=0（子宫），得 tianfuIdx=0（子宫），天府紫微同宫——明显不对。

**正确算法：** 天府+紫微宫位索引之和 = 4（mod 12）
```js
tianfuIdx = (4 - ziweiIdx + 12) % 12
```

**验证（标准对照表）：**
| 紫微宫位 | 子(0) | 丑(1) | 寅(2) | 卯(3) | 辰(4) | 巳(5) | 午(6) | 未(7) | 申(8) | 酉(9) | 戌(10) | 亥(11) |
|---------|-------|-------|-------|-------|-------|-------|-------|-------|-------|-------|--------|--------|
| 天府宫位 | 辰(4) | 卯(3) | 寅(2) | 丑(1) | 子(0) | 亥(11)| 戌(10)| 酉(9) | 申(8) | 未(7) | 午(6)  | 巳(5)  |

规律：两宫地支索引之和 = 4（mod 12）。寅寅同宫（寅+寅=4），午戌差=4。

## Bug #2 — 紫微系空宫偏移错误（影响太阳/廉贞/武曲/天同宫位）

**错误代码（旧 v2.0）：**
```js
stars['太阳'] = (ziweiIdx - 2 + 12) % 12;
stars['廉贞'] = (ziweiIdx - 5 + 12) % 12;
```

**问题：** 忘记了空宫跳跃。紫微系口诀中有"隔一""隔二位"的空格。

**正统口诀：** 紫微天机逆行旁，**隔一**阳武天同当，又**隔二位**廉贞地  
（天机逆1，空1，太阳逆3，武曲逆4，天同逆5，空3宫，廉贞逆9——注意空宫跳跃）

**正确代码（v3.0）：**
```js
stars['天机'] = (ziweiIdx - 1 + 12) % 12;  // 逆1
stars['太阳'] = (ziweiIdx - 3 + 12) % 12;  // 逆3（跳过1空宫）
stars['武曲'] = (ziweiIdx - 4 + 12) % 12;  // 逆4
stars['天同'] = (ziweiIdx - 5 + 12) % 12;  // 逆5
stars['廉贞'] = (ziweiIdx - 8 + 12) % 12;  // 逆8（跳过2空宫）
```

**注意争议点：** 廉贞的偏移量各派有-8和-9两种说法。  
v3.0 采用 -8（梁若瑜飞星派 + 付老师一致），如需验证，用已知案例比对。

## Bug #3 — 文昌/文曲按年支安（错误），应按生时安

**错误代码：**
```js
// 旧代码用年支(yearZhiIdx)计算文昌文曲
result['文昌'] = { palaceIdx: (10 - yearZhiIdx + 12) % 12 };
result['文曲'] = { palaceIdx: (4 + yearZhiIdx) % 12 };
```

**问题：** 文昌文曲是按"生时"安，不是年支。  
口诀：**文昌戌宫起子时逆数，文曲辰宫起子时顺数**

**正确代码（v3.0）：**
```js
result['文昌'] = { palaceIdx: (10 - shichenIdx + 12) % 12 };  // 戌(10)起逆数至生时
result['文曲'] = { palaceIdx: (4 + shichenIdx) % 12 };        // 辰(4)起顺数至生时
```

## Bug #6 — 火星/铃星只用年支，未考虑生时

**旧代码：** 仅按年支对照表直接返回固定宫位（忘记顺数生时）

**正统算法：** 年支三合局 → 起始宫，再从起始宫顺数至生时

```js
// 火星：起宫 + 生时顺数
const fireStart = HUOXING_START[yearZhiIdx] ?? 2;
result['火星'] = { palaceIdx: (fireStart + shichenIdx) % 12 };

// 铃星：寅午戌→卯(3)起，其余→戌(10)起，加生时顺数
const bellStart = LINGXING_START[yearZhiIdx] ?? 10;
result['铃星'] = { palaceIdx: (bellStart + shichenIdx) % 12 };
```

**火星起宫对照：**
- 寅午戌年 → 丑宫(1)起
- 申子辰年 → 寅宫(2)起
- 巳酉丑年 → 卯宫(3)起
- 亥卯未年 → 酉宫(9)起

## Bug #7 — 五行局用"年干简化法"（影响紫微定宫及整个命盘）★ v3.0 修正

**错误代码：**
```js
// ❌ 用年干简化，大量命盘五行局算错
const wuxingJu = { '甲':5,'己':5, '乙':4,'庚':4, '丙':2,'辛':2, '丁':3,'壬':3, '戊':6,'癸':6 }[yearGan] || 3;
```

**问题：**  
年干法是近似值，在很多命盘中与纳音法结果不同。  
付老师案例验证：1975乙卯年，年干乙→金四局(4)，但命宫壬午纳音→木三局(3)。  
两者不同，用年干法时**紫微星位置完全算错**（子宫 vs 午宫），整个命盘全错。

**正确算法：命宫纳音五行局**（付老师第31节明确示范）

```
步骤：
  1. 先算命宫（calcMingPalace）
  2. 安宫干（calcPalaceGan），得命宫宫干
  3. 命宫宫干 + 命宫地支 → 查纳音得五行局数

天干配数：甲乙=1, 丙丁=2, 戊己=3, 庚辛=4, 壬癸=5
地支配数：子丑午未=1, 寅卯申酉=2, 辰巳戌亥=3
干支数相加，大于5则减5，余数：1→木三, 2→金四, 3→水二, 4→火六, 5→土五
```

```js
// ✅ 正确：命宫纳音五行局
function calcWuxingJuNaYin(mingGan, mingDz) {
    const tgNum = {'甲':1,'乙':1,'丙':2,'丁':2,'戊':3,'己':3,'庚':4,'辛':4,'壬':5,'癸':5};
    const dzNum = {'子':1,'丑':1,'午':1,'未':1,'寅':2,'卯':2,'申':2,'酉':2,'辰':3,'巳':3,'戌':3,'亥':3};
    const juMap = {1:3, 2:4, 3:2, 4:6, 5:5};
    let sum = tgNum[mingGan] + dzNum[mingDz];
    if (sum > 5) sum -= 5;
    return juMap[sum] || 3;
}
```

**验证：** 1975乙卯年命宫壬午 → 壬=5, 午=1, 5+1=6>5 → 6-5=1 → 木三局(3) ✅

## Bug #8 — 紫微定宫用 ceil(day/ju) 简化算法（影响不整除的日子）★ v3.0 修正

**错误代码：**
```js
// ❌ 用 ceil 简化，不整除日子结果错误
const groupIdx = Math.ceil(lunarDay / wuxingJu) - 1;
const ziweiIdx = (startPalace + groupIdx) % 12;
```

**问题：**  
`Math.ceil` 对"只加不减到整除，阳退阴进"逻辑的近似处理，与付老师算法有出入。  
实测：木三局17日 → ceil 法算到子宫，付老师算法算到**午宫**，差了6宫！

**正确算法：**
```js
// ✅ 付老师"只加不减到整除，阳退阴进"
function calcZiweiPosition(lunarDay, wuxingJu) {
    let d = lunarDay, added = 0;
    while (d % wuxingJu !== 0) { d++; added++; }
    const quotient = d / wuxingJu;
    let zhiIdx = (2 + quotient - 1) % 12;  // 从寅(2)起数商数格
    if (added > 0) {
        zhiIdx = (added % 2 === 1)
            ? (zhiIdx - added + 120) % 12   // 单数→退
            : (zhiIdx + added) % 12;         // 偶数→进
    }
    return zhiIdx;
}
```

**验证：** 木三局17日 → 17+1=18, 18/3=6, 寅起数6=未, 添加1单→退1=午 ✅
